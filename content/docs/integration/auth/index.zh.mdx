Ask Xuan 只使用 [Supabase Auth](https://supabase.com/auth) 作为认证提供方。浏览器登录、OAuth 跳转、用户会话和 token 刷新都由 Supabase 处理。应用会把已登录的 Supabase 用户同步到本地 `user` 资料表，让后台管理、RBAC、积分、支付和订阅继续使用现有 Drizzle 模型。

认证是可选的。如果未配置 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY`，应用会以静态模式运行，并隐藏登录页。

## 前置条件

在配置认证之前，请确保已完成：

1. Supabase 项目 — 参考[数据库配置](/docs/integration/database/supabase)。
2. 如果需要用户资料、后台管理、RBAC、支付、积分或订阅功能，请配置运行时数据库连接 `DATABASE_URL`。

## 邮箱密码认证

邮箱密码登录由 Supabase Auth 处理。密码策略、邮箱确认、SMTP 和邮件模板请在 Supabase Dashboard 中配置。

登录表单会调用 `supabase.auth.signInWithPassword`。注册表单会调用 `supabase.auth.signUp`，并把用户姓名写入 Supabase user metadata。

## OAuth 社交登录

OAuth Provider 在 Supabase 中配置，不再通过本应用环境变量保存 provider secret。GitHub、Google 或其他登录方式请在 `Authentication` → `Sign In / Providers` 中启用。

每个 provider 使用 Supabase Dashboard 中展示的 callback URL。应用会通过 `@supabase/ssr` 获取 Supabase 管理的浏览器会话 Cookie。

## 管理员访问

管理员用户可以访问后台管理面板，管理用户、支付、积分和配置。

将用户的邮箱添加到 `ADMIN_EMAILS` 环境变量即可授予管理员权限：

```bash
ADMIN_EMAILS=admin@example.com,another-admin@example.com
```

多个邮箱用英文逗号分隔。

## 配置参考

| 环境变量 | 必填 | 说明 |
|---------|------|------|
| `VITE_SUPABASE_URL` | 是 | Supabase 项目 URL |
| `VITE_SUPABASE_ANON_KEY` | 是 | 用于浏览器认证的 Supabase publishable/anon key |
| `DATABASE_URL` | 否 | 本地用户资料和业务数据使用的运行时 PostgreSQL 连接 |
| `ADMIN_EMAILS` | 否 | 管理员邮箱，多个用逗号分隔 |
| `VITE_TURNSTILE_CAPTCHA_ENABLED` | 否 | 启用 Turnstile 验证码（`true`/`false`） |
| `VITE_TURNSTILE_SITE_KEY` | 否 | Turnstile Site Key |
| `TURNSTILE_SECRET_KEY` | 否 | Turnstile Secret Key |

## 数据库表

Supabase Auth 会把凭据和会话存储在 Supabase `auth` schema。Ask Xuan 在现有 public 表中保存应用资料和业务数据：

| 表名 | 说明 |
|------|------|
| `auth.users` | Supabase 管理的认证用户 |
| `user` | 从 Supabase Auth 同步的应用用户资料 |
| `user_role` | RBAC 角色分配 |
| `credits`、`order`、`payment`、`subscription` | 关联应用用户资料的业务数据 |
