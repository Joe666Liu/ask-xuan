Ask Xuan uses [Supabase Auth](https://supabase.com/auth) as the only authentication provider. Browser login, OAuth redirects, user sessions, and token refresh are handled by Supabase. The app mirrors the signed-in Supabase user into the local `user` profile table so admin, RBAC, credits, payments, and subscriptions can keep using the existing Drizzle models.

Authentication is optional. If `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` are not configured, the app runs in static mode and the login page is hidden.

## Prerequisites

Before setting up authentication, make sure you have:

1. A Supabase project — see [Database](/docs/integration/database/supabase) for setup instructions.
2. A runtime database connection in `DATABASE_URL` if you need user profiles, admin, RBAC, payments, credits, or subscriptions.

## Email/Password Authentication

Email/password login is handled by Supabase Auth. Configure password policy, email confirmation, SMTP, and templates in the Supabase Dashboard.

The login form calls `supabase.auth.signInWithPassword`. Sign-up calls `supabase.auth.signUp` and stores the user's name in Supabase user metadata.

## OAuth Providers

OAuth providers are configured in Supabase, not in this app's environment variables. Add GitHub, Google, or other providers under `Authentication` → `Sign In / Providers`.

Use the callback URL shown in the Supabase Dashboard for each provider. The app then receives the Supabase-managed browser session cookies through `@supabase/ssr`.

## Admin Access

Admin users have access to the admin panel for managing users, payments, credits, and configuration.

To grant admin access, add the user's email to the `ADMIN_EMAILS` environment variable:

```bash
ADMIN_EMAILS=admin@example.com,another-admin@example.com
```

Multiple emails are separated by commas.

## Configuration Reference

| Environment Variable | Required | Description |
|---------|------|------|
| `VITE_SUPABASE_URL` | Yes | Supabase project URL |
| `VITE_SUPABASE_ANON_KEY` | Yes | Supabase publishable/anon key used by browser auth |
| `DATABASE_URL` | No | Runtime PostgreSQL connection for local user profiles and business data |
| `ADMIN_EMAILS` | No | Comma-separated admin email addresses |
| `VITE_TURNSTILE_CAPTCHA_ENABLED` | No | Enable Turnstile captcha (`true`/`false`) |
| `VITE_TURNSTILE_SITE_KEY` | No | Turnstile Site Key |
| `TURNSTILE_SECRET_KEY` | No | Turnstile Secret Key |

## Database Tables

Supabase Auth stores credentials and sessions in the Supabase `auth` schema. Ask Xuan keeps application profiles and business data in the existing public tables:

| Table | Description |
|------|------|
| `auth.users` | Supabase-managed authenticated users |
| `user` | Application profile mirrored from Supabase Auth |
| `user_role` | RBAC role assignments |
| `credits`, `order`, `payment`, `subscription` | Business data linked to the application profile |
