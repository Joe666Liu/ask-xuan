[Supabase](https://supabase.com/) is built on PostgreSQL and provides authentication, file storage, Edge Functions, and more beyond the database. Check the [Supabase pricing page](https://supabase.com/pricing) for current free-tier limits.

<Callout type="info" title="Integration scope">
Ask Xuan uses Supabase as a PostgreSQL provider through Drizzle ORM. Authentication remains powered by Better Auth, and file uploads use the project's S3-compatible storage layer.
</Callout>

## Quick Start

<div className="fd-steps">
  <div className="fd-step">
    ### Create a Project
    Open the [Supabase Dashboard](https://supabase.com/dashboard) and click `New Project`.
    <Callout type="warn" title="Tip">
    Remember the database password you set — you'll need it later when connecting.
    </Callout>
    ![Supabase Create Project](/docs/integration/database/supabase-create.webp)
  </div>
  <div className="fd-step">
    ### Get the Connection String
    After the project is created, click the `Connect` button at the top.
    ![Supabase Project Link](/docs/integration/database/supabase-connect.webp)

    Select `ORMs` → `Drizzle`, and copy the connection string in `.env` format. Replace `[YOUR-PASSWORD]` with the password you set.
    ![Supabase Project Link](/docs/integration/database/supabase-env.webp)
  </div>
  <div className="fd-step">
    ### Configure Environment Variables
    Paste the pooled transaction connection string into your `.env` file for application runtime. It usually uses port `6543`.

    ```bash
    DATABASE_URL="postgresql://postgres.xxx:password@aws-0-region.pooler.supabase.com:6543/postgres"
    BETTER_AUTH_SECRET="your-generated-secret"
    ```

    For migrations, prefer Supabase's direct connection string. Add it as `DATABASE_MIGRATION_URL`; Drizzle Kit will use it automatically and fall back to `DATABASE_URL` when it is not set.

    ```bash
    DATABASE_MIGRATION_URL="postgresql://postgres:password@db.xxx.supabase.co:5432/postgres"
    ```

    If your local network does not support IPv6 and you do not have Supabase's IPv4 add-on, use the session pooler URL on port `5432` for `DATABASE_MIGRATION_URL`.
  </div>
  <div className="fd-step">
    ### Run Migrations
    Run the following commands to generate and apply database migrations:

    ```bash tab="pnpm"
    pnpm db:generate
    pnpm db:migrate
    ```
    ```bash tab="npm"
    npm run db:generate
    npm run db:migrate
    ```
    ```bash tab="bun"
    bun db:generate
    bun db:migrate
    ```
  </div>
  <div className="fd-step">
    ### Manage Database
    Use Drizzle Studio to view and edit data:
    ```bash tab="pnpm"
    pnpm db:studio
    ```
    ```bash tab="npm"
    npm run db:studio
    ```
    ```bash tab="bun"
    bun db:studio
    ```
  </div>
</div>

## Using a Separate Schema

Supabase uses the `public` schema by default, which contains Supabase's own system tables (auth, storage, etc.). Creating a separate schema keeps your application data isolated from system data for easier management.

Create a new schema in the Table Editor page:

![Supabase Schema](/docs/integration/database/supabase-schema-create.webp)

Then specify the schema in the connection string:

```bash
DATABASE_URL="postgresql://postgres.xxx:password@aws-0-region.pooler.supabase.com:6543/postgres?schema=your-schema-name"
DATABASE_MIGRATION_URL="postgresql://postgres:password@db.xxx.supabase.co:5432/postgres?schema=your-schema-name"
```

## Notes

- Runtime uses `DATABASE_URL` through `postgres.js` with prepared statements disabled, which is compatible with Supabase transaction pooling.
- Drizzle migrations use `DATABASE_MIGRATION_URL` when present, so schema changes can use Supabase's direct Postgres connection.
- The app does not depend on Supabase Data API for server-side queries, so newly created table exposure settings in Supabase Data API do not affect normal application reads and writes.
